**自动化部署流程**:

```bash
# 1. 开发者提交代码
git push origin main

# 2. GitHub Actions 自动触发
# - 运行 ci-cd.yml 流水线
# - 执行代码检查、测试、构建

# 3. Docker 镜像构建和推送

# - 推送到 GitHub Container Registry

# 4. 自动部署到生产环境
# - SSH 连接到服务器
# - 拉取最新镜像
# - 停止并删除旧容器
# - 启动新容器

# 5. 健康检查
# - 检查 /health 端点
# - 发送飞书通知
```

**本地部署命令** (备用方案):

```bash
# 构建并运行
docker-compose up -d --build

# 查看日志
docker-compose logs -f app

# 重启服务
docker-compose restart app

# 停止服务
docker-compose down
```

**PM2 部署命令** (备用方案):

```bash
# 安装依赖
npm ci --production

# 构建前端
npm run build

# 使用 PM2 启动

pm2 status
```
