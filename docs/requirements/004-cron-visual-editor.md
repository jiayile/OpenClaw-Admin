## 一、功能描述

### 1.1 背景

当前 OpenClaw-Admin 的定时任务 (Cron) 配置功能存在以下问题：

1. **Cron 表达式学习门槛高**：用户需要了解 `* * * * * *` 各字段的含义
2. **配置容易出错**：手动输入表达式容易写错，无实时校验
3. **缺乏可视化预览**：无法直观看到下次执行时间
4. **无试运行能力**：配置后只能等待实际执行验证

用户反馈显示，约 40% 的定时任务问题源于 Cron 表达式配置错误。

### 1.2 目标

- 提供可视化的 Cron 表达式生成器
- 支持多种配置模式（简单/高级）
- 实时预览下次执行时间
- 提供表达式校验和试运行功能
- 降低用户配置门槛，减少错误率

### 2.2 核心功能

#### 2.2.1 常用模板
预设常用 Cron 表达式模板：

| 模板名称 | Cron 表达式 | 说明 |
|---------|-------------|------|
| 每分钟 | `* * * * * *` | 每分钟执行一次 |
| 每小时 | `0 * * * * *` | 每小时第 0 分钟执行 |
| 每天凌晨 | `0 0 0 * * *` | 每天 00:00:00 执行 |
| 每天固定时间 | `0 0 9 * * *` | 每天 09:00:00 执行 |
| 每 5 分钟 | `*/5 * * * * *` | 每 5 分钟执行一次 |
| 每 30 分钟 | `0 */30 * * * *` | 每 30 分钟执行一次 |
| 每周一 | `0 0 9 * * 1` | 每周一 09:00:00 执行 |
| 每月 1 号 | `0 0 9 1 * *` | 每月 1 号 09:00:00 执行 |

#### 2.2.2 表达式校验
- 实时语法校验
- 错误提示和修复建议
- 无效表达式阻止保存

#### 2.2.3 执行预览
- 显示未来 5 次执行时间
- 支持自定义预览时间范围
- 时区感知（使用用户本地时区）

#### 2.2.4 试运行功能
- 手动触发一次任务执行（不保存调度）
- 查看试运行结果
- 验证任务逻辑正确性

### 4.3 关键实现

#### Cron 表达式解析

```typescript
import { parseExpression } from 'cron-parser';

function getNextRuns(cronExpression: string, count: number = 5): Date[] {
  try {
    const interval = parseExpression(cronExpression);
    const runs: Date[] = [];
    
    for (let i = 0; i < count; i++) {
      const next = interval.next();
      runs.push(next.toDate());
    }
    
    return runs;
  } catch (error) {
    throw new Error(`Invalid cron expression: ${error.message}`);
  }
}
```

#### 表达式校验

```typescript
function validateCronExpression(expression: string): {
  valid: boolean;
  error?: string;
} {
  try {
    parseExpression(expression);
    return { valid: true };
  } catch (error) {
    return { valid: false, error: error.message };
  }
}
```

## 五、UI/UX 设计要点

### 5.1 设计原则

1. **渐进式披露**：简单模式满足 80% 需求，高级模式供专家使用
2. **即时反馈**：任何配置变更立即显示结果
3. **错误预防**：通过 UI 约束减少错误输入
4. **清晰表达**：Cron 表达式含义用自然语言解释

### 5.2 交互流程

```
用户打开定时任务创建页面
         ↓
选择"可视化编辑器"
         ↓
┌──────────────┬──────────────┐
│   简单模式    │   高级模式    │
└──────────────┴──────────────┘
         ↓              ↓
选择频率模板    手动配置各字段
         ↓              ↓
显示预览结果    显示预览结果
         ↓              ↓
    合并 → 试运行验证
         ↓
       保存任务
```

## 九、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Cron 表达式复杂性超出预期 | 开发延期 | 先实现核心功能，边缘情况后续迭代 |
